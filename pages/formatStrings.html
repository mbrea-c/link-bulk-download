Format Strings

Format strings provide a powerful way to name your downloaded files, allowing the use of information specific to each file in the saved filename by means of placeholder variables. The placeholder variables in your format string will be replaced by their corresponding value for each file you download.

As an instructive example, consider the format string "My file is named {name} and was the {index}th file selected". If we assume that you selected the links pointing to "https://example.com/foo" and "https://example.com/bar" in that order, the downloaded files would be named:

  • My file is named foo and was the 1th file selected
  • My file is named bar and was the 2th file selected

Available placeholder variables

  • {index} — Index of the selected link, in order of selection. That is, the first link selected will have an index of 1, the second will have an index of 2, etc.
  • {name} — Basename of the selected link. This is the last segment of the URI path, stripped away of the query and fragment components. E.g., the URI "https://example.com/foo?q=test#somefragment" would have basename foo.
  • {linkText} — Visible text on the selected link. This is (most likely) what firefox names your downloaded item when you choose "Save Link As..." by default.

Note: More placeholder variables will be added soon.
